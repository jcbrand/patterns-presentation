Why do designers like to work with prototypes?
  • Building to think
  • Learning faster by failing
  • Permission to explore new behaviors

The value of first making a rich, interactive prototype.

  • Communicates the designer's vision to client and rest of team.
  • Initial user-testing and feedback can be gathered.
  • Client knows from the start what they'll be getting, UI and UX-wise.
  • Assumptions can be tested before they're implemented.
  • Design problems are solved during the planning stage.
  • Developers see the context within which their work will be exposed to the end-user.
  • Demonstrations make it easier to sell a website, concept or idea.
So, what is* Patterns? * not a grammar mistake, talking about the toolkit here.

Patterns is a toolkit which allows designers to create rich, interactive prototypes in HTML/CSS without having to learn or write any Javascript

From a designer's perspective

Pick and choose individual patterns, to add functionality and interactivity to your prototype.


More complex behavior can be achieved by combining different patterns.

From a developer's perspective, Patterns lets you write, encapsulate and then expose front-end Javascript features to designers in a way that comes naturally to them.

Stuff like:

  • Toggling or switching CSS classes
  • Tabs
  • Ajax page loading
  • Rich text editing
  • Live cloning of page elements
  • Form validation
  • ... Pretty much anything that can be done in JS
